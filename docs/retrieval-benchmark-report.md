## 分析

### 表现良好的场景

- **单术语/单词查询**（`Cache`、`存储`、`考研`）：Recall = 1.0，jieba + ILIKE 精确命中
- **精确术语查询**（`计算机系统概述`、`计算机组成原理`）：Recall = 1.0，自定义词典正确保留整词

### P@5 偏低的原因

P@5 = 0.367 主要受语料库规模影响：11 条 EvidenceUnit 中期望数仅 1-4 条，最大 Precision 受 `1/5 = 0.2` 约束。语料库扩大到数百条后 P@5 将更具参考价值。

### `Cache 存储原理` Recall = 0.5 的原因

jieba 分词为 `['Cache', '存储', '原理']`，三词 AND 查询过于严格。「Cache 存储原理」对应的 EvidenceUnit 中只包含「Cache」而非「原理」。后续可通过降低多词查询的匹配门槛（AND → OR + 排序加权）改善。
