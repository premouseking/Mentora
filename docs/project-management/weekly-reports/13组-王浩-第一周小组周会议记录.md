### 1.3 产品 UX 设计与前端实现

**建课流程：**
- 输出桌面产品 UX 设计文档及概念图
- 设计并实现建课确认流程（课程界面、SetupPages、SetupContinuationPages）
- 设计课程工作台、学习任务页面、阶段总结和方案调整页面
- 实现错题集、上划栏、资料库等交互组件

**建课流程重设计 v2（进行中）：**
- 流程从 4 步扩展为 5 步：描述目标 → 补充信息 → 资料上传 → 信息追问 → 生成学习方案
- 描述目标和补充信息从自由文本改为固定选项模式
- 资料上传独立成页，可选跳过，上传后 AI 自动解读
- 信息追问由 AI 主导（PlannerAgent），左侧气泡引导交互
- 底栏从「浮层仅展示信息」升级为「最终学习方案」
- 新增组件：CourseCreationContext（共享状态）、AiMessageBubble（AI 引导气泡）、CourseInfoBar（常驻底栏）
- 设计规格文档已输出（`2026-06-18-course-creation-redesign.md`）

### 1.4 AI 对话面板

- 前端接入流式 SSE，通过 `fetch + ReadableStream` 实时逐字解析显示
- 后端实现 `POST /api/chat/stream/` SSE 视图（WSGI 兼容同步桥接）
- Vite 配置代理 `/api → 127.0.0.1:8000`，打通前后端联调
- DeepSeek API 真实调用验证通过，AI 面板逐字流式输出可用

### 1.5 Agent 运行时与模型网关

- 模型网关模块落地（FakeProvider 测试驱动 + OpenAIProvider 自建 SSE 客户端）
- Agent 运行时模块落地：Orchestrator（编排器）、TutorAgent（问答 + 工具调用循环）、PlannerAgent（学习计划生成）、ClarifierAgent（意图澄清）
- 基础设施：ContextManager（Token 预算控制）、ToolRegistry（工具注册）、EventEmitter（事件回调）
- 25+ 集成测试通过（FakeProvider 驱动，无需外部 LLM）
- 前后端流式 SSE 联调验收通过

### 1.6 资料解析与检索

**资料解析核心链路（M2-M5）：**

| 模块 | 交付内容 | 状态 |
|------|---------|------|
| M2 | ParsedBundle 与 EvidenceUnit 契约 | 已落地 |
| M3 | 文本 PDF 解析 Worker（PyMuPDF Adapter） | 已落地 |
| M4 | 解析基准数据集 | 进行中 |
| M5 | Evidence 持久化与 pgvector 检索 | 已落地 |

**检索增强全链路（E1-E4）：**
- E1：Embedding Provider 抽象层 + 豆包 Embedding 集成（MRL 降维 2048→1024d）
- E2：Chunk Embedding 生成 Celery 任务（批量调用，幂等跳过已有 Embedding）
- E3：FTS + Trgm + Vector 三路 RRF 混合检索融合（权重 0.5/0.2/0.3）
- E4：向量搜索基准对比 + 延迟降级修复（无 API key 时 3.3s 超时 → <1ms 降级）

**附加交付：**
- 表格提取 POC（PyMuPDF `find_tables()`）
- 多列 PDF 阅读顺序恢复
- OCR 预研方案记录（暂缓执行）
- 前端全局资料库页面（文件夹管理、拖拽上传）
