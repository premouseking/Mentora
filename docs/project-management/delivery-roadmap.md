# 阶段交付路线图

## 推进原则

- 项目按照能力成熟度划分阶段，不预设固定工期。
- 当前阶段退出标准全部通过后，才进入下一阶段。
- WH 负责组织阶段验收，并结合四位成员的验收证据决定是否进入下一阶段。
- 第一版面向 Windows，并要求联网。
- 后端保持 Django 模块化单体。
- 第一阶段只支持文本 PDF。
- 每个阶段结束时必须完成一次可运行的端到端演示。

## 第一阶段：工程基线与资料处理链路

**阶段目标：** 用户选择 PDF、完成上传、观察真实后台处理状态，并看到明确的成功或失败结果。

| 成员 | 主要交付 |
| --- | --- |
| WH | Django/Celery/PostgreSQL/Redis 基线、上传 API、处理状态、SSE、阶段集成 |
| LBZ | 上传和工作流进度 UI、API Client、React Query 规范 |
| LH | PyMuPDF Spike、ParsedBundle 和 EvidenceUnit Schema、解析 Worker |
| LWJ | 模型网关骨架、结构化输出校验器、确定性 Fake Provider |

集成门槛：

```text
Renderer -> 上传 API -> 对象存储 -> Celery 解析
         -> 持久化状态 -> SSE/REST -> Renderer
```

退出标准：

- 文件正文不通过 JSON 或 IPC 复制；
- 重复完成上传请求具有幂等性；
- Worker 失败能够进入终态；
- 至少一个真实 PDF 完成端到端处理；
- 页面刷新后能够恢复最终处理状态；
- 本地安装和验证命令已形成文档。

## 第二阶段：基于证据的课程问答

**进入条件：** 第一阶段退出标准全部通过。

**阶段目标：** 将已处理 PDF 加入课程，提出问题，并打开回答引用的原文位置。

| 成员 | 主要交付 |
| --- | --- |
| WH | 资料和作用域持久化、激活事务、运行时事件、权限校验 |
| LBZ | 资源库、课程资料选择、Tutor 回答、引用和原文查看 UI |
| LH | Evidence 持久化、pgvector 和关键词检索、RRF、引用定位 |
| LWJ | 受 Evidence 约束的 Tutor 回答生成和引用校验 |

退出标准：

- 课程当前作用域外的资料不会进入召回结果；
- 回答引用稳定的 Evidence ID 和页码；
- 用户可以从回答跳转到正确原文位置；
- 证据不足时明确拒答；
- 基准问题能够输出召回率和引用准确率。

## 第三阶段：课程创建与计划确认

**进入条件：** 第二阶段问答链路稳定，作用域越权测试全部通过。

**阶段目标：** 描述学习目标、回答澄清问题、检查画像和计划，然后开始学习。

| 成员 | 主要交付 |
| --- | --- |
| WH | 建课状态机、画像和计划版本、原子启动、并发控制 |
| LBZ | 需求输入、画像确认、计划确认、校验和启动 UI |
| LH | 为计划输入提供资料目录和证据摘要 |
| LWJ | 画像候选、澄清策略、计划生成和校验 |

退出标准：

- 未确认候选不能成为 active 事实；
- confirmed 画像不可变；
- 不合法计划不能进入 `ready_to_start`；
- 修改画像后旧计划不能启动；
- 画像和计划在同一事务中启用。

## 第四阶段：完整学习闭环

**进入条件：** 第三阶段能够稳定创建并启动课程。

**阶段目标：** 继续学习、完成任务、进行追问，并获得下一任务。

| 成员 | 主要交付 |
| --- | --- |
| WH | LearningTask、LearningSession、LearningEvent 和状态迁移服务 |
| LBZ | 课程工作台、任务页、Tutor 交互、阶段总结 |
| LH | 会话证据快照和作用域检索 |
| LWJ | 学习包生成、Tutor 行为、阶段总结和下一任务策略 |

退出标准：

- 每个学习包固定画像、计划、作用域和证据版本；
- 完成任务会产生不可变事件；
- 页面刷新和应用重启后能够恢复当前学习状态；
- 下一任务决策可以解释；
- 阶段总结不会静默修改学习计划。

## 第五阶段：基础测评闭环

**进入条件：** 学习任务、事件和证据快照能够稳定运行。

**阶段目标：** 完成一组短测验，并获得带证据的评分反馈。

| 成员 | 主要交付 |
| --- | --- |
| WH | 题目版本、表单、Attempt、Response、评分和幂等 next-item |
| LBZ | 测验、作答、反馈和问题报告 UI |
| LH | 出题证据快照和作用域准入 |
| LWJ | 答案优先生成、确定性门槛、盲解和歧义检查 |

退出标准：

- 支持单选、判断、确定性短答案和数值题；
- 硬门槛失败的题目不会展示；
- 同一 Attempt 位置不能创建重复题目；
- 隔离问题题目后保留历史，并可撤销掌握度证据；
- 模型或验证器失败时存在可靠降级。

## 第六阶段：桌面发布候选版

**进入条件：** 建课、学习和测评三条核心链路全部通过端到端验收。

**阶段目标：** 安装、登录、完成核心流程，重启应用后恢复状态。

| 成员 | 主要交付 |
| --- | --- |
| WH | Main/Preload 安全、认证、SSE Bridge、打包、NSIS、更新和发布决策 |
| LBZ | 桌面体验完善、无障碍和 Playwright Electron E2E |
| LH | 大文件稳定性、解析性能和基准回归 |
| LWJ | 模型质量回归、预算、超时和 Fallback 调优 |

退出标准：

- Renderer 无法访问 Node.js、Token、任意路径或任意 URL；
- 安装包通过完整核心流程 Smoke Test；
- 应用重启后恢复云端拥有的状态；
- 发布元数据晚于二进制发布；
- 已知风险和延期能力形成文档；
- WH 和三位领域负责人共同确认发布候选版。

## 首个里程碑之后再做

- 超出有限 Spike 的扫描 PDF OCR；
- 完整 Office、音频和视频处理；
- 正式考试或认证考试；
- 无约束开放题评分；
- IRT 或强化学习策略；
- 离线 AI 和离线检索；
- 多人协作；
- 插件、通用浏览器、本地 Agent Runtime 和微服务。
