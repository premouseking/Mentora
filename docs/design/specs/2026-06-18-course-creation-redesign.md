## 1. 新流程

```text
描述目标 → 补充信息 → 资料上传 → 信息追问 → 生成学习方案
```

**五步，底栏逐步填充为最终学习方案。**

### 1.1 描述目标（步骤 1）

- 固定选项，不接入 AI
- 选项示例：准备考试、系统学习一个领域、完成项目实践、快速入门……
- 每个选项点击后展开子选项（如考试日期、科目等）
- 用户填完 → 信息写入底栏（标题：「学习目标」）

### 1.2 补充信息（步骤 2）

- 固定选项，不接入 AI
- 当前基础（完全新手 / 了解基础 / 学过一遍 / 不确定）
- 可投入方式（集中多学 / 短时间推进 / 稳定节奏 / 暂不确定）
- 用户填完 → 信息写入底栏（标题：「当前基础」「推进方式」等）

### 1.3 资料上传（步骤 3）

- **独立步骤，可选跳过**
- 支持上传文件（PDF、图片、Word 等），需接入 AI 进行内容识别与解读
- 上传完成后 AI 自动解读文件内容，提取学习相关结构化信息
- 解读结果写入底栏（标题：「参考资料」）
- 提供「跳过」入口，不阻塞后续流程
- **设计理由**：上传与 AI 解读需要耗时，独立成页提供自然的等待态；且 AI 解读后在信息追问阶段能基于资料提出更精准的问题

### 1.4 信息追问（步骤 4）

- **AI 主导**，由 PlannerAgent 根据前序步骤信息（含资料解读结果）生成追问
- **中间主区域**：AI 生成的问答交互（选择题 / 填表 / 输入框），每一轮一个问题
- **左侧**：AiMessageBubble，展示 AI 的简短引导描述，帮助用户理解当前问题
  - 气泡横向固定宽度（≤ 左边空白区域），纵向随内容撑开
  - 尾巴垂直居中，指向 Mentora 图标（Sparkles）
- 追问持续到 AI 判断信息足够生成方案
- 每轮回答 → 信息写入底栏（逐项追加）

### 1.5 生成学习方案（步骤 5）

- 从步骤 4 点击「下一步」进入
- **动画**：
  1. 底栏从收起态（1/5 高度）自然展开到 ~2/3 高度
  2. 展开到位后，底栏向上滑动，停在页面标题下方
  3. 此时底栏**不再贴底**，成为页面内的方案展示区
- 底栏内容变为只读的最终学习方案（与收集过程中填充的结构一致）
- 内部可滚动，无可见滚动条
- 底部显示确认按钮

## 2. 底栏角色

| 阶段 | 底栏状态 | 内容 |
|------|---------|------|
| 步骤 1-3 | 收起态（1/5 高度），固定底部 | 已填入的条目 |
| 步骤 4 | 同上，逐步追加新条目 | 追问中逐项填入 |
| 步骤 5 动画前 | 收起态 → 展开到 2/3 | 完整信息 |
| 步骤 5 动画后 | 展开态，不贴底，页面内展示 | 最终学习方案（只读） |

底栏的视觉设计需要更丰富的排版，不只是 key-value 表格：
- 字段按类别分组（目标类 / 约束类 / 资料类 / 系统建议）
- 组间有分隔，组内紧凑
- 可以用小图标注来源

## 3. AiMessageBubble 规格

- 位置：左侧空白区域
- 宽度：`max-width: 260px`，不超过左边留白
- 样式：圆角气泡 + 左侧三角尾巴指向圆形 Mentora 图标
- 尾巴：CSS `::before` / `::after` 双三角（外框 + 填充色）
- 高度：`flex` 随内容撑开
- 内容：简短引导文字（1-3 句）

## 4. 与旧版差异

| | 旧版 | 新版 |
|---|---|---|
| 步骤数 | 5 步（描述 → 澄清 → 资料 → 需求 → 方案） | 5 步（描述 → 补充 → 资料上传 → 追问 → 方案） |
| AI 介入时机 | 步骤 2 起逐步介入 | 步骤 3（资料解读）+ 步骤 4（集中追问） |
| 底栏内容 | 浮层仅展示已了解信息 | 角色升级为「学习方案」，最终成为确认页主内容 |
| 描述目标 | 自由文本 | 固定选项 |
| 资料添加 | 独立步骤 | 独立步骤（含 AI 解读，可跳过） |
| 确认需求 | 独立步骤 | 融入信息追问 |
| 方案确认 | overlay 面板 | 底栏展开后不贴底，页面内展示 |
