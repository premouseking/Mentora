# ADR-0004：课程画像与学习路径采用双阶段确认

- 状态：Proposed
- 日期：2026-06-12
- 关联设计：`docs/architecture/end-to-end-implementation-plan.md`

## 背景

模型澄清用户需求的目的，是填写结构化课程配置，而不是直接决定用户画像。即使画像
字段正确，系统自动生成的学习路径仍可能在顺序、时长、学习方式和取舍上不符合用户
预期。若澄清结束后直接启动计划，用户无法在执行前检查这些决定。

## 决策

1. 模型澄清只产生 `CourseProfileCandidate`，并填充可编辑画像草稿。
2. `extraction_confidence` 只保存在候选记录中，不进入用户确认后的正式画像字段。
3. 用户点击“生成学习方案”时冻结 `CourseProfileRevision` 为 `confirmed`，但不激活
   课程。
4. 计划器只读取不可变的 confirmed 画像、精确知识作用域、Topic 修订和学习快照。
5. 计划以 `Phase -> Unit -> TaskTemplate` 卡片生成，并保存为可编辑
   `LearningPlanRevision` 草稿。
6. 计划层只允许修改计划结构。截止时间、总预算、范围、目标深度等变化必须回到画像
   草稿，并重新生成计划。
7. 计划通过依赖、预算、覆盖率和资料可用性校验后才能进入 `ready_to_start`。
8. 用户点击“开始学习”时，在同一事务中激活画像和计划；禁止只激活其中一方。
9. 首次启动前不物化可执行任务；已有课程调整时，旧活动计划继续服务到替代计划确认。
10. 已有课程仅调整路径时可以沿用当前 active 画像，但应用事务必须验证画像指针未变；
    修改画像时必须原子切换画像和计划两个指针。

## 状态机

```text
collecting_requirements
  -> reviewing_profile
  -> generating_plan
  -> reviewing_plan
  -> active
```

计划不可行时停留在 `reviewing_plan`，用户可以调整路径，或返回画像面板修改目标和
约束。任何返回画像的修改都会使基于旧画像的计划草稿变为 `needs_replan`。

## 结果

正向结果：

- 用户明确拥有画像和学习路径的最终决定权；
- 模型置信度不会污染已经确认的业务事实；
- 可以在执行前发现前置冲突、预算超限和覆盖不足；
- 画像与路径通过精确修订关联，可以复现每次规划依据；
- 最终激活具备原子性，不出现画像与路径版本错配。

代价：

- 建课增加一个计划检查步骤；
- 需要画像候选、画像修订和计划修订三类对象；
- 计划卡片编辑需要依赖和预算重新校验；
- 返回修改画像时必须重新生成或局部重算计划。

## 验收不变量

1. 未经用户确认的候选值不能成为 active 画像事实。
2. confirmed 画像不可修改，只能克隆成新草稿。
3. 计划必须记录精确 `profile_revision_id` 和知识作用域修订。
4. 未进入 `ready_to_start` 的计划不能产生可执行任务。
5. 改变画像事实后，旧计划草稿不能继续启动。
6. “开始学习”不能只激活画像或只激活计划。
7. 模型提取置信度不参与用户已经确认字段的后续规划权重。
