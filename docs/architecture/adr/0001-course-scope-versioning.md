# ADR-0001：课程知识作用域版本管理

- 状态：Proposed
- 日期：2026-06-11
- 关联设计：`docs/architecture/scope-versioning-design.md`

## 背景

用户可以持续上传、替换和停用学习资料，并在课程进行期间调整资料范围。系统需要同时
满足：

- 新资料不会静默改变课程；
- 历史回答能够追溯当时使用的资料和范围；
- 只分析增量变化，避免每次全库比较；
- 多窗口编辑和后台任务不能造成丢失更新；
- 用户可以恢复历史配置，但不能破坏审计链。


## 决策

采用“完整作用域快照 + 持久差异 + 不可变证据快照”。

1. 文件内容变化创建不可变 `SourceVersion`。
2. 课程资料调整创建 `CourseKnowledgeScopeRevision` 草稿。
3. 每个作用域修订保存完整 `CourseScopeBinding` 集合。
4. Binding 使用跨修订稳定的 `binding_key`，并持久化 `BindingDiff`。
5. 草稿通过 `lock_version` 和 `If-Match` 乐观锁编辑。
6. 影响分析固定 Binding checksum、课程配置版本、Topic 版本和分析器版本。
7. 激活时锁定课程行，在单一事务内切换 active 指针并写 Outbox。
8. 检索服务只读取服务端解析出的 active 作用域。
9. 每次模型调用写入不可变 `TurnEvidenceSnapshot`。
10. 恢复历史配置时创建新修订，不直接重新激活历史修订。

M0 仅支持“上传新资料”和“为已有资料上传新版本”两个明确入口，不实现智能版本合并、
会话临时 Overlay、章节自动迁移和复杂 Claim 冲突。

## 选择理由

- 作用域快照只复制关联元数据，存储成本远低于正文、解析产物和向量；
- 运行时直接读取完整快照，查询和故障恢复比增量事件重放简单；
- 持久差异仍可支持影响分析、用户展示和审计；
- PostgreSQL 的部分唯一约束、行锁、事务和 Outbox 足以支撑首版；
- 资料版本与课程选材版本分离后，同一资料可以安全复用于多门课程。

## 拒绝方案

### 直接修改当前 Binding

无法追溯历史边界，分析期间容易发生丢失更新，也无法可靠恢复历史状态。

### 只保存增量事件

需要在每次检索前重放事件或维护额外投影，恢复、校验和迁移成本高于首版收益。

### Git 式文件补丁和分支合并

课程作用域主要是结构化关联数据，不是文本协作仓库。Git 语义无法直接解决权限、
解析状态、Topic 影响和学习计划一致性。

### 新资料自动覆盖旧资料

系统无法仅凭文件名或相似度判断教师口径、考试范围和版本权威性，静默覆盖风险不可接受。

## 结果

正向结果：

- 历史回答和引用可复现；
- 检索边界由后端强制执行；
- 资料增量分析有明确输入；
- 可以安全处理并发激活和恢复历史配置。

代价：

- 每次正式调整会增加一组 Binding 快照记录；
- 需要维护状态机、校验和、影响报告及一致性 Reconciler；
- 新旧资料范围迁移失败时需要用户确认。

## 实施前验证门槛

1. PostgreSQL 约束能保证每门课程最多一个 active 修订。
2. 两个客户端并发编辑时，过期 `If-Match` 稳定返回 412。
3. 两个草稿基于同一旧 active 修订时，最多一个能够成功激活。
4. Binding 变化后旧影响报告不能激活。
5. 未绑定资料无法被检索接口命中。
6. 历史回答能够定位作用域修订、资料版本和 EvidenceUnit。
7. 软删除资料不会破坏历史引用，仍被引用的对象不能物理清理。
8. 角色或优先级变化不会触发文件重解析和无关 Embedding 重建。

## 待评审问题

- M0 的 PDF 选择器是否只支持全文，还是同时支持页码范围；
- 废弃草稿的保留期限；
- 官方资料失去平台授权后，历史引用允许展示到什么粒度；
- 一门课程允许的最大 Binding 数和单次影响分析预算。
