## 三、课程页上划栏（/courses/{id}）

### 布局

```
┌──────┬──────────────────────────────┬──────────┐
│      │   树状图（从左到右展开）       │  侧边栏   │
│ 阶   │                              │          │
│ 段   │  Phase → Chapter → Task      │  当前选中  │
│ 选   │   ●───●────●                  │  节点的   │
│ 择   │   ●───●────●────●             │  详细解释  │
│ 按   │   ●───●────●                  │          │
│ 钮   │                              │          │
│      │                              │          │
└──────┴──────────────────────────────┴──────────┘
```

- **左侧**：纵向阶段选择按钮（窄栏），当前阶段高亮，已完成打 ✅
- **中间**：横向树状图，Phase → Chapter → Task
- **右侧**：侧边栏，根据选中节点展示解释信息

### 侧边栏内容

| 选中节点 | 侧边栏展示 |
|----------|-----------|
| Phase | 阶段目标、预估总时长、权重占比 |
| Chapter | 章节说明、知识点概览 |
| Task | 知识点详解、相关例题、关联资料 |

## 四、阶段类型池（8 种）

| 类型 | 适用场景 |
|------|---------|
| 基础入门 | 零基础起步，核心概念和基本方法 |
| 知识梳理 | 系统学习教材，按章节深入 |
| 专项训练 | 突破薄弱点，题型分类训练 |
| 综合应用 | 跨知识点实战，真题/项目 |
| 实战演练 | 限时模拟，成套练习 |
| 查漏补缺 | 错题回看，高频错误强化 |
| 考前冲刺 | 考点回顾，临场策略 |
| 拓展提升 | 课外延伸，竞赛/前沿方向 |

### AI 回应用户调整

- 用户可增删、调序、改名
- 调整完后 AI 根据新阶段列表重新生成 Chapter

## 五、AI 分阶段与权重指导（重要）

不同学习档案应有不同的阶段分配策略。通过 prompt 引导 AI 输出合理的阶段组合和权重。

### 典型档案 → 建议阶段序列 + 权重

| 档案特征 | 建议阶段 | 权重倾向 |
|----------|---------|---------|
| **零基础冲刺考试**（有截止、零基础） | 基础入门 → 专项训练 → 考前冲刺 | 60% / 25% / 15% |
| **有基础冲刺考试**（有截止、中等基础） | 知识梳理 → 专项训练 → 综合应用 → 考前冲刺 | 20% / 30% / 30% / 20% |
| **系统学习无截止** | 基础入门 → 知识梳理 → 综合应用 → 拓展提升 | 20% / 40% / 25% / 15% |
| **快速入门**（兴趣导向、零基础） | 基础入门 → 专项训练 → 实战演练 | 50% / 30% / 20% |
| **考前突击**（临近截止、有基础） | 查漏补缺 → 考前冲刺 | 40% / 60% |
| **补弱专项** | 专项训练 → 综合应用 | 70% / 30% |

### Prompt 策略入门

AI 在生成阶段列表时应遵循：

1. **先看截止日期**：有截止且紧迫 → 压缩基础阶段，加大冲刺/查漏权重
2. **再看当前基础**：零基础 → 基础入门权重高，不得跳过
3. **再看学习目标**：考试类 → 必含考前冲刺；兴趣类 → 可含拓展提升
4. **最后看时间预算**：每天时间少 → 减少阶段数（3-4 个）；时间充足 → 可增至 5 个
