## 一、入口与导航

- **进入方式**：课程页上划栏底部「调整方案」按钮 → `adjustMode = true` 切换面板
- **退出方式**：左上角 ↶ 返回按钮 → 回到三栏学习计划预览
- **模式区分**：
  - `adjustMode = false`：左上角显示「学习计划」，X 关闭按钮
  - `adjustMode = true`：左上角显示「调整学习方案」，↶ 返回按钮

## 三、AI 对话栏（第一部分）交互流程

### 3.1 ChatStage 状态机

```
  input ──[发送]──→ options ──[选择选项]──→ (等待按钮)
                          │
  档案编辑完成 ──────────→ ai-thinking ──[2s后]──→ ai-result
```

类型定义：
```ts
type ChatStage = "input" | "options" | "ai-thinking" | "ai-result";
```

### 3.2 阶段一：input（输入阶段）

- **标题**：「调整学习方案」黑体加粗居中（20px）
- **副标题**：「与 AI 讨论你想如何调整学习方案」灰色居中（14px）
- **输入框**：多行 textarea（min 80px / max 160px），边框 `1.5px solid var(--border-strong)`，圆角 10px
- **发送按钮**：右下角（输入框外部），绿色 `#059669`，空内容时变灰 `#d1d5db`
- **键盘**：Enter 发送（Shift+Enter 换行）

### 3.3 阶段二：options（选项选择）

- **子标题**：「新的学习时长安排」（14px bold）
- **6 个时间选项**：flex wrap 两列排布（一行两个）
  - 每天 1 小时以内
  - 每天 1～2 小时
  - 每天 2～3 小时
  - 每天 3～4 小时
  - 每天 4 小时以上
  - 不固定，根据进度灵活安排
- **选中态**：绿色边框 + 浅绿背景
- **两个操作按钮**（flex 横向占满）：
  - 「确认并生成学习方案」（primary）
  - 「确认并继续调整」（secondary）

### 3.4 阶段三：ai-thinking（AI 思考）

- 复用 `MentoraLoader` 组件，显示「AI 正在分析你的档案调整…」
- 自动 2 秒后切换到 ai-result

### 3.5 阶段四：ai-result（AI 结果）

- 结果卡片：边框圆角框，展示 AI 回复文本（mock）
- 同两个操作按钮
- **当前为 mock 数据**，真实 AI 对话待后续接入

## 四、学习档案表格（第二部分）

### 4.1 数据结构

```ts
interface ProfileItem {
  key: string;
  title: string;
  value: string;
}
```

Mock 字段：学习目标、当前基础、推进方式、每日时长、目标日期、学校/地区

### 4.2 查看 / 编辑模式

| 元素 | 查看模式 | 编辑模式 |
|------|---------|---------|
| 右上角按钮 | ✎（Pencil 图标） | ✕（取消）+ ✓（确认） |
| 表格内容列 | 纯文本 | `<input>` 输入框（绿色边框） |
| ✕ 按钮 | — | 取消编辑，恢复原始值 |
| ✓ 按钮 | — | 完成编辑 → 触发 ai-thinking → ai-result |

### 4.3 编辑完成流程

点击 ✓ → `setProfileEditing(false)` → `setChatStage("ai-thinking")` → 2s 后 → `setChatStage("ai-result")`

### 4.4 取消编辑

点击 ✕ → `setProfileEditing(false)` → `setProfileValues(MOCK_PROFILE)`（恢复原始值，不触发 AI 思考）
