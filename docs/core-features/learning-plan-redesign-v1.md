## 一、与初版方案的核心差异

| 初版 | v1 重设计 |
|------|-----------|
| Phase → Unit → TaskTemplate 三层 | Phase → Chapter → Task 三层（概念对齐教材） |
| 确定性调度器 + 时长估计器 | 砍掉——AI 直接分阶段，不伪精确 |
| 可行性检查 + 降级方案 | 砍掉——建课诉求是「看一眼合理就开干」 |
| Unit 绑定 Topic + 前置依赖 | 章节按内容顺序排列，不做形式化依赖声明 |
| 卡片编辑器（拖拽/拆分/合并） | 不提供——用户调整方式为简化/加强 |

## 二、学习计划的两种展示场景

### 场景 A：建课结尾（确认页）

**目标**：让用户看到 AI 生成的完整路线，审视后拍板。

**设计思路**：保留当前 Phase 卡片式概览——阶段列表 + 目标 + 占比 + 代表性任务 + 简化/加强按钮。

不展示细节的 Chapter / Task 树，避免信息过载。用户在这个阶段的诉求是「能不能接受」，不是「每章内容对不对」。

### 场景 B：课程页上划栏

**目标**：查看学习进度，知道当前在哪、下一步是什么。

**布局**：

```
┌──────┬──────────────────────────────┬──────────┐
│      │   树状图（从左到右展开）       │  侧边栏   │
│ 阶   │                              │  解释区   │
│ 段   │  Phase → Chapter → Task      │          │
│ 选   │   ●───●────●                  │  选中节   │
│ 择   │   ●───●────●────●             │  点的详   │
│ 按   │   ●───●────●                  │  细说明   │
│ 钮   │                              │          │
│      │                              │          │
└──────┴──────────────────────────────┴──────────┘
```

**左侧**：纵向排列的阶段选择按钮（窄栏），当前阶段高亮。

**中间**：横向树状图，Phase 是根节点 → Chapter 是枝 → Task 是叶。根节点在左，叶子在右。

**右侧**：侧边栏。根据选中内容展示不同信息：
- 选中 Phase → 阶段目标、总时长、占比
- 选中 Chapter → 章节说明、预计学习时长、主要知识点
- 选中 Task → 知识点详解、相关例题、关联资料

## 三、Phase（阶段）

阶段是计划的第一级结构。AI 根据学习档案生成 4-5 个阶段，不固定模板。

**典型阶段示例**：

| 阶段 | 说明 |
|------|------|
| 基础梳理 | 系统掌握核心概念和基础知识 |
| 专项突破 | 针对重点/难点集中强化训练 |
| 综合应用 | 跨章节综合，真题实战 |
| 考前冲刺 | 高频考点回顾 + 模拟测试 |

阶段数量不固定——用户目标不同，AI 自由生成。

---

## 四、Chapter（章节）

### 命名

| 中文 | 英文 | 理由 |
|------|------|------|
| 章节 | Chapter | 与教材/课程教材分节方式一致，用户直觉理解 |

### 各阶段的章节划分

| 阶段 | 章节划分依据 | 示例 |
|------|-------------|------|
| 基础梳理 | 教材章节 / 主题 | 第1章 集合、第2章 函数、第3章 数列 |
| 专项突破 | 题型 / 考点分类 | 选择题技巧、证明题思路、计算题速解 |
| 综合应用 | 综合卷 / 专题 | 历年真题卷 A、模拟卷 B、跨章节综合 |
| 考前冲刺 | 高频考点 / 易错点 | 高频考点 Top10、易错题集、考前模拟 |

每个阶段的章节划分依据和粒度由 AI 按学习档案决定，不强制统一格式。

---

## 五、Task（任务）

任务是章节下的知识点枚举，是学习内容的叶子节点。

**作用**：告诉用户在这一章要学哪些知识点，不作为完成追踪单元。进度追踪在 Phase 级别。

**示例**（"第2章 函数" → Tasks）：

- 函数定义与表示法
- 定义域与值域
- 函数单调性
- 函数奇偶性
- 函数图像变换
