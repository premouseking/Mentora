# 建课流程：初始方案 vs 实际实现

> 记录核心概念差异，方便对照初始设计理解当前实现。
> 初始方案详见 `docs/architecture/end-to-end-implementation-plan.md`。

---

## 一、概念重塑

### 初始方案

```
用户画像 → 确认画像 → 学习方案 → 确认方案 → 激活
```

三个独立环节，每个需用户单独确认。

### 实际实现

```
学习档案 ─┐
           ├── 一次性确认 → 开始学习
学习计划 ─┘
```

**核心变化**：

| 初始概念 | 实际概念 | 说明 |
|----------|----------|------|
| 用户画像 | **学习档案** | 不再单独确认，融入基础信息 + 追问两个步骤 |
| 学习方案 | **学习档案 + 学习计划** | 拆分为两部分，档案是"已知信息"，计划是"阶段安排" |
| 画像确认 | 无 | 去除独立步骤，画像要素打散到基础信息和追问中 |

---

## 二、步骤对比

### 初始方案（3 步确认）

```
步骤 1：描述目标
步骤 2：AI 追问 → 用户画像 → 确认画像
步骤 3：生成学习方案 → 确认方案 → 激活
```

### 实际实现（5 步）

| 步骤 | 页面 | 收集内容 → 归属 |
|------|------|-----------------|
| 1 | 描述目标 | 学习目标（自然语言） → 学习档案 |
| 2 | 基础信息 | 基础/推进方式/时间/学校 → 学习档案 |
| 3 | 资料上传 | 关联资料（可选跳过） → 学习档案 |
| 4 | 信息追问 | AI 追问，收集更多信息 → 学习档案 |
| 5 | 确认方案 | **学习档案** + **学习计划** → 一次性确认 |

**关键差异**：

- 步骤 2 替代了「画像确认」——基础信息直接构成学习档案的一部分
- 步骤 4 追问结果也归入学习档案（存储为「补充信息」条目）
- 步骤 5 把档案和计划放在同一页确认，不再分两步

---

## 三、学习档案的构成

学习档案在步骤 5 确认页显示，包含以下条目：

| 条目 | 来源步骤 | 说明 |
|------|----------|------|
| 课程名称 | 步骤 5（AI 生成） | 排在档案最前面 |
| 学习目标 | 步骤 1 | 用户原始描述 |
| 基础信息 | 步骤 2 | 基础/推进方式/时间/学校/截止日期 |
| 上传资料 | 步骤 3 | 关联的资料文件名列表 |
| 补充信息 | 步骤 4 | AI 追问的全部问答汇总 |

---

## 四、学习计划的含义

学习计划是 AI 生成的阶段性方案，由多个**学习阶段**组成。

每个阶段包含：
- 阶段名称和目标
- 相对学习量（百分比）
- 代表性任务列表
- 可手动调整（简化 / 加强）

**与初始方案的差异**：

- 初始方案：方案为一个大块的「学习方案」
- 实际实现：拆分为「学习档案」+「学习计划」两部分，档案是静态信息，计划是动态的阶段安排

---

## 五、追问环节

- AI 追问随时可以停止，直接跳到确认方案页
- 追问结果汇总为「补充信息」，归入学习档案
- 最多 8 轮（初始方案最多 3 轮）

## 六、后端状态机

```
DRAFT → REFINING → GENERATING_PLAN → COMPLETED → STARTED
```

与前端 5 步对应关系：

| 状态 | 前端步骤 |
|------|----------|
| DRAFT | 步骤 1-2（描述目标 + 基础信息） |
| REFINING | 步骤 3-4（资料上传 + 追问） |
| GENERATING_PLAN | 步骤 5（AI 生成方案中） |
| COMPLETED | 步骤 5（方案已生成，等待确认） |
| STARTED | 确认后，开始学习 |

## 七、学习计划：文档设计 vs 当前实现

| 方面 | 文档设计（§9） | 当前实际 |
|------|---------------|----------|
| **生成方式** | 确定性调度器计算 + AI 翻译 | AI 直接生成（PlannerAgent 一把梭） |
| **结构深度** | Phase → Unit → TaskTemplate 三层 | Phase 一层为主，Unit 自动占位 |
| **单元绑定** | Unit 绑定 Topic，有前置依赖、目标深度 | 无 topic 绑定、无前置依赖 |
| **任务** | 结构化 TaskTemplate（类型/模式/分钟/必修） | 纯文本字符串，自动转为默认 lecture |
| **可行性** | 三态 + 降级方案选项 | 只判断 feasible/constrained，无降级 UX |
| **激活** | 用户确认后激活 → 再物化任务 | 生成后立即自动激活 |
| **编辑器** | 卡片式拖拽编辑 + 依赖检查 | 只有「简化/加强」两个按钮 |
| **任务物化** | 按 2 周窗口动态物化 | 同设计，已实现 `materialize_tasks()` |

## 八、学习方案展示：文档设计 vs 当前实现

### 概述

课程工作区上划栏（PhaseSummary）的重设计，从单层列表改为三栏结构。

### 初始方案

```
┌────────────────────────────┐
│  方案概览（阶段数/任务数）    │
│  关键指标（4 个卡片）        │
│  阶段安排（纵向列表）        │
│  首阶段单元详览              │
│  下一阶段预览                │
│  [调整计划] [开始学习]       │
└────────────────────────────┘
```

单栏纵向滚动，所有信息线性排列，无阶段间快速跳转，无任务级别详情。

### 实际实现（v6 设计 → v7 落地）

```
┌──────┬───────────────────────┬───────────┐
│      │                       │           │
│ 阶   │  主干 + 分支画板       │  详情     │
│ 段   │                       │  基础信息  │
│ 导   │  [任务]─┐             │  概述     │
│ 航   │         ├─ 章节1      │  相关资料  │
│ 栏   │  [任务]─┘             │           │
│      │         ┌─[任务]      │ [开始练习] │
│      │ 章节2 ──┤             │           │
│      │         └─[任务]      │           │
│      │                       │           │
├──────┴───────────────────────┴───────────┤
│              [调整计划]                    │
└──────────────────────────────────────────┘
```

**核心变化**：

| 初始概念 | 实际概念 | 说明 |
|----------|----------|------|
| 方案概览 + 阶段列表 | **左栏纵向阶段导航** | 阶段间快速切换，六边形箭头块 + 双向 snap 滚动 |
| 单元列表（Phase → Unit → Task） | **中栏主干分支画板** | Phase → Chapter → Task 三层结构可视化，章节居中轴，任务左右交替分支 |
| 无独立详情区 | **右栏详情栏** | 选中节点后展示基础信息（键值对）+ 概述（拼句）+ 相关资料（按钮列表） |
| 下拉关闭 | **X 按钮关闭** | 下拉手势与中栏滚动冲突，保留 X 按钮 |

### 结构对比

| 维度 | 初始（v0） | 实际（v7） |
|------|-----------|-----------|
| **布局** | 单栏纵向滚动 | 三栏（左 180px + 中 flex + 右 280px） |
| **阶段导航** | 列表中被动浏览 | 滚轮/点击主动跳转，至多 4 个可见，双向 snap |
| **章节展示** | 折叠列表项 | 主干节点 + SVG 箭头连线（表示学习顺序） |
| **任务展示** | 文字行（类型+时长） | 分支节点 + SVG"丰"字形连线，左右交替 |
| **节点详情** | 无 | 点击任意节点 → 右栏显示差异化信息 |
| **任务区分** | 统一展示 | 讲解→「知识点」+ 知识名；练习→「练习N」+ 开始练习按钮 |
| **交付方式** | delivery_mode 直译 | 按任务类型重定义（讲解→自主确认，练习→完成练习） |
| **数据** | 连后端 ActivePlan | 全 mock（5 阶段 11 单元，含 knowledge_point / materials） |

### 阶段导航细节

| 特性 | 说明 |
|------|------|
| **箭头形状** | 六边形（6 顶点 clip-path），上凹下凸，凸尖朝下，相邻块 margin-top:-14px 嵌合 |
| **滚动规则** | 向下：选中停上起第 3 位（index-2）；向上：选中停下起第 3 位（index-1） |
| **滚动实现** | `overflow: hidden` + 纯 JS `scrollTo`（非原生滚动），用实测 `offsetTop` 定位 |
| **点击行为** | 未选中阶段→切换+显示阶段信息；已选中→仅切回阶段信息 |

### 画板细节

| 特性 | 说明 |
|------|------|
| **布局算法** | `computeLayout(units)` 预计算坐标，DOM `position:absolute` + SVG 覆盖层共享坐标系 |
| **连线方案** | SVG `<line>` + `<marker>` 向下箭头（章节间主干），分支"丰"字形无箭头 |
| **任务左右** | 偶数章节右侧、奇数章节左侧，同章节多任务纵向排列 |
| **防碰撞** | 章节高度 = `max(章节高, 任务数×任务高)` |
| **滚动** | `overflow: auto` 原生滚轮/滚动条（未实现按住拖拽） |

### 详情栏细节

| 选中节点 | 基础信息字段 | 概述 | 额外 |
|----------|-------------|------|------|
| 阶段 | 序号、章节数、任务数、时长 | objective 原文 | — |
| 章节 | 序号、任务数、目标深度、时长 | 拼句（章节含 N 任务，涵盖类型） | — |
| 讲解任务 | 类型="知识点"、交付="自主确认"、时长、必修 | 拼句（学习 X 知识点） | 相关资料按钮 |
| 练习任务 | 类型="练习"、交付="完成练习"、时长、必修 | 拼句（巩固知识点） | 相关资料按钮 + **开始练习按钮** |

### 实现要点

- **父组件 overlay 手势隔离**：overlay 的 `setPointerCapture` 会劫持 click，左栏加 `onPointerDown stopPropagation` 阻断
- **类型声明合并坑**：`courseApi.ts` 有两个同名 `export interface PlanPhase`，组件用本地 `NavPhase`（仅 id+title）规避
- **滚动防抖**：中/右栏均加 `scrollbar-gutter: stable`
- **设计文档**：`docs/core-features/learning-plan-redesign-v6.md`（设计）、`v7.md`（实现落地）
