# 建课流程：初始方案 vs 实际实现

> 记录核心概念差异，方便对照初始设计理解当前实现。
> 初始方案详见 `docs/architecture/end-to-end-implementation-plan.md`。

---

## 一、概念重塑

### 初始方案

```
用户画像 → 确认画像 → 学习方案 → 确认方案 → 激活
```

三个独立环节，每个需用户单独确认。

### 实际实现

```
学习档案 ─┐
           ├── 一次性确认 → 开始学习
学习计划 ─┘
```

**核心变化**：

| 初始概念 | 实际概念 | 说明 |
|----------|----------|------|
| 用户画像 | **学习档案** | 不再单独确认，融入基础信息 + 追问两个步骤 |
| 学习方案 | **学习档案 + 学习计划** | 拆分为两部分，档案是"已知信息"，计划是"阶段安排" |
| 画像确认 | 无 | 去除独立步骤，画像要素打散到基础信息和追问中 |

---

## 二、步骤对比

### 初始方案（3 步确认）

```
步骤 1：描述目标
步骤 2：AI 追问 → 用户画像 → 确认画像
步骤 3：生成学习方案 → 确认方案 → 激活
```

### 实际实现（5 步）

| 步骤 | 页面 | 收集内容 → 归属 |
|------|------|-----------------|
| 1 | 描述目标 | 学习目标（自然语言） → 学习档案 |
| 2 | 基础信息 | 基础/推进方式/时间/学校 → 学习档案 |
| 3 | 资料上传 | 关联资料（可选跳过） → 学习档案 |
| 4 | 信息追问 | AI 追问，收集更多信息 → 学习档案 |
| 5 | 确认方案 | **学习档案** + **学习计划** → 一次性确认 |

**关键差异**：

- 步骤 2 替代了「画像确认」——基础信息直接构成学习档案的一部分
- 步骤 4 追问结果也归入学习档案（存储为「补充信息」条目）
- 步骤 5 把档案和计划放在同一页确认，不再分两步

---

## 三、学习档案的构成

学习档案在步骤 5 确认页显示，包含以下条目：

| 条目 | 来源步骤 | 说明 |
|------|----------|------|
| 课程名称 | 步骤 5（AI 生成） | 排在档案最前面 |
| 学习目标 | 步骤 1 | 用户原始描述 |
| 基础信息 | 步骤 2 | 基础/推进方式/时间/学校/截止日期 |
| 上传资料 | 步骤 3 | 关联的资料文件名列表 |
| 补充信息 | 步骤 4 | AI 追问的全部问答汇总 |

---

## 四、学习计划的含义

学习计划是 AI 生成的阶段性方案，由多个**学习阶段**组成。

每个阶段包含：
- 阶段名称和目标
- 相对学习量（百分比）
- 代表性任务列表
- 可手动调整（简化 / 加强）

**与初始方案的差异**：

- 初始方案：方案为一个大块的「学习方案」
- 实际实现：拆分为「学习档案」+「学习计划」两部分，档案是静态信息，计划是动态的阶段安排

---

## 五、追问环节

- AI 追问随时可以停止，直接跳到确认方案页
- 追问结果汇总为「补充信息」，归入学习档案
- 最多 8 轮（初始方案最多 3 轮）

## 六、后端状态机

```
DRAFT → REFINING → GENERATING_PLAN → COMPLETED → STARTED
```

与前端 5 步对应关系：

| 状态 | 前端步骤 |
|------|----------|
| DRAFT | 步骤 1-2（描述目标 + 基础信息） |
| REFINING | 步骤 3-4（资料上传 + 追问） |
| GENERATING_PLAN | 步骤 5（AI 生成方案中） |
| COMPLETED | 步骤 5（方案已生成，等待确认） |
| STARTED | 确认后，开始学习 |

## 七、学习计划：文档设计 vs 当前实现

| 方面 | 文档设计（§9） | 当前实际 |
|------|---------------|----------|
| **生成方式** | 确定性调度器计算 + AI 翻译 | AI 直接生成（PlannerAgent 一把梭） |
| **结构深度** | Phase → Unit → TaskTemplate 三层 | Phase 一层为主，Unit 自动占位 |
| **单元绑定** | Unit 绑定 Topic，有前置依赖、目标深度 | 无 topic 绑定、无前置依赖 |
| **任务** | 结构化 TaskTemplate（类型/模式/分钟/必修） | 纯文本字符串，自动转为默认 lecture |
| **可行性** | 三态 + 降级方案选项 | 只判断 feasible/constrained，无降级 UX |
| **激活** | 用户确认后激活 → 再物化任务 | 生成后立即自动激活 |
| **编辑器** | 卡片式拖拽编辑 + 依赖检查 | 阶段级编辑表格：拖拽排序（grip 按钮）+ 添加/删除阶段 + 锁定只读 |
| **任务物化** | 按 2 周窗口动态物化 | 同设计，已实现 `materialize_tasks()` |

## 八、学习方案展示：文档设计 vs 当前实现

### 概述

课程工作区上划栏（PhaseSummary）的重设计，从单层列表扩展为两个模式：**预览模式**（三栏结构）和 **调整模式**（四区布局）。

### 模式切换

| 模式 | 触发条件 | 顶部标题 | 按钮 |
|------|----------|----------|------|
| 预览模式 | 默认进入 | 学习计划 | X 关闭 |
| 调整模式 | 底部「调整方案」按钮 | 调整学习方案 | ↶ 返回 |

### 预览模式：三栏布局（v6→v7）

```
┌──────┬───────────────────────┬───────────┐
│      │                       │           │
│ 阶   │  主干 + 分支画板       │  详情     │
│ 段   │                       │  基础信息  │
│ 导   │  [任务]─┐             │  概述     │
│ 航   │         ├─ 章节1      │  相关资料  │
│ 栏   │  [任务]─┘             │           │
│      │         ┌─[任务]      │ [开始练习] │
│      │ 章节2 ──┤             │           │
│      │         └─[任务]      │           │
│      │                       │           │
├──────┴───────────────────────┴───────────┤
│              [调整计划]                    │
└──────────────────────────────────────────┘
```

| 维度 | 说明 |
|------|------|
| **布局** | 三栏（左 180px + 中 flex + 右 280px） |
| **阶段导航** | 滚轮/点击主动跳转，至多 4 个可见，双向 snap |
| **章节展示** | 主干节点 + SVG 箭头连线（表示学习顺序） |
| **任务展示** | 分支节点 + SVG"丰"字形连线，左右交替 |
| **节点详情** | 点击任意节点 → 右栏显示差异化信息 |
| **任务区分** | 讲解→「知识点」+ 知识名；练习→「练习N」+ 开始练习按钮 |
| **交付方式** | 按任务类型重定义（讲解→自主确认，练习→完成练习） |
| **数据** | 全 mock（5 阶段 11 单元，含 knowledge_point / materials） |

### 调整模式：四区布局 + 三区变更追踪（v8→v9）

```
┌───────────────────────────────────────────────┐
│  调整学习方案                          [↶]   │
├───────────────────────────────────────────────┤
│  1️⃣ AI 对话栏                                │
│  ┌─────────────────────────────────────────┐  │
│  │    调整学习方案 / 选项 / 思考 / 结果     │  │
│  ├─────────────────────────────────────────┤  │
│  │  [提示小字]                   [发送/按钮] │  │
│  └─────────────────────────────────────────┘  │
│                                               │
│  2️⃣ 学习档案  *                       ↩ ✎   │
│  ┌──────┬─────────────────────────────────┐  │
│  │ 项目 │ 内容                            │  │
│  └──────┴─────────────────────────────────┘  │
│                                               │
│  3️⃣ 学习计划概览  *                    ↩ ✎  │
│  ┌───────────────────────────────────────┐   │
│  │ [1 基础][2 知识][3 专项][4 …] ← 滑轨  │   │
│  ├───────────────────────────────────────┤   │
│  │  [第 1 阶段] 基础入门                 │   │
│  │  掌握集合、函数等核心概念与基本方法    │   │
│  │  章节 3 章  任务 12 个  预估 8 小时   │   │
│  └───────────────────────────────────────┘   │
│                                               │
├───────────────────────────────────────────────┤
│          [  确认方案  ]  绿色/灰色            │
└───────────────────────────────────────────────┘
```

#### 三区变更追踪

三个独立 dirty 标志追踪各自的修改状态：

| 区域 | 状态变量 | 触发条件 |
|------|----------|----------|
| AI 对话栏 | `chatChanged` | 输入非空 / 选中选项 / 发送 |
| 学习档案 | `profileChanged` | ✓ 确认编辑 |
| 学习计划 | `planChanged` | ✓ 确认编辑 |

- 任一变更为 true → 标题后红色 `*`，提示小字动态显示"已说明需求；已更改学习档案；已更改学习计划。发送给AI以确认更改。"
- 两个确认按钮（「确认并生成学习方案」「确认并继续调整」）→ 调用 `clearAllDirty()` 清除所有 dirty
- 底部「确认方案」按钮：`planGenerated && !profileChanged && !planChanged` 时才绿色可点

#### 学习档案编辑

| 条件 | 标题 | 按钮 |
|------|------|------|
| 未编辑 + 未 dirty | 学习档案 | 🖉 |
| dirty | 学习档案 * | ↩ + 🖉 |
| 编辑中 | 学习档案 | ✕ + ✓ |

- ✓ 确认 → 退出编辑 + `profileChanged = true`，不再触发 AI 思考
- ✕ 取消 → 恢复原始值 + 退出编辑
- ↩ 撤回 → 恢复原始值 + 清除 dirty

#### 学习计划编辑

**概览模式**（`!planEditing`）：阶段导航滑轨 + 章节画布（PlanOverviewCanvas）+ 阶段/章节详情栏

- 滑轨：平行四边形阶段块，滚轮纵向切换（方向感知 snap），横向 touch-pad 正常滚动，`overscroll-behavior: contain` 防泄漏
- 画布：章节块横向排列，SVG 箭头连线，点击切换详情
- 详情栏：阶段模式（基础信息+概述）或章节模式（基础信息+概述+相关资料）

**阶段编辑表格**（`planEditing = true`，v9 新增）：

| planEditing | planChanged | 左侧按钮 | 右侧按钮 | 表格行为 |
|:--:|:--:|--|--|--|
| ❌ | ❌ | — | 🖉 | 概览模式 |
| ❌ | ✅ | ↩ | 🖉 | 概览模式 |
| ✅ | ❌ | ✕ | ✓ | **编辑模式：可拖拽 + 可添加** |
| ✅ | ✅ | ↩ | 🖉 | **锁定模式：只读表格** |

编辑表格特性：
- **左右两栏**：阶段名称 + 简介描述，flex 比例 1:2 对齐表头
- **拖拽排序**：最右 `⋮⋮` grip 按钮（仅 grip 可拖拽，点其他位置无效），HTML5 Drag & Drop，拖拽中行半透明
- **+ 添加阶段**：底部满宽虚线按钮 → 弹出可选阶段列表（MOCK_PHASE_POOL 6 个，已选过滤）
- **锁定模式**：✓ 确认后表格保持可见但只读（无 grip、无拖拽、无 +），点击 🖉 重新解锁编辑

#### 底部确认按钮

- 固定底部，`planGenerated && !profileChanged && !planChanged` 时绿色可点
- 否则灰色 `disabled`
- 点击 → 退出调整模式，回到三栏预览

## 九、建课流程：5 步 → 2 步（v10）

### 概述

建课流程从 5 步简化为 2 步：**建立档案 → 确认方案**，去除冗余的中间确认环节。

### 步骤对比

| 旧流程（5 步） | 新流程（2 步） |
|---------------|---------------|
| 描述目标 | → 建立学习档案（内含 4 个子步骤） |
| 基础信息 | ↗ |
| 资料上传 | ↗ |
| 信息追问 | ↗ |
| 确认方案 | → 确认学习方案 |

### 步骤 1：建立学习档案（BuildProfilePage）

路径 `/courses/new`，4 个子步骤：

| 子步骤 | 类型 | 内容 |
|--------|------|------|
| 1 | input | 学习方向：你想学习什么？ |
| 2 | materials | 学习资料：选择资料 + 右上角「上传资料」按钮（弹出和资源库相同的上传弹窗） |
| 3 | choice | 学习方式：6 选项（视频/阅读/练习/实践/讨论/辅导） |
| 4 | choice | 时间安排：6 选项（30 分钟/1 小时/2 小时/工作日/周末/灵活） |

- 底部双按钮：**确认并生成学习方案**（→ 步骤 2）、**确认并继续完善档案**（→ 下一子步骤）
- 第 1 步两按钮均灰色，第 1 步「生成方案」不可用（必须有学习目标）
- 最后一步只显示「确认并生成学习方案」，占满横向
- 按钮固定底部（SetupShell footer prop）

### 调整方案重新生成态

路径 `/courses/new?adjust=true`，从确认方案页「调整方案」按钮进入：
- 问题描述："已收到你的更改。是否重新生成学习方案？"
- 内容区：输入框（placeholder: "描述你希望如何调整学习方案…"）
- 底部双按钮均可用（不再受 step1 限制）

### 步骤 2：确认学习方案（ConfirmPlanPage）

路径 `/courses/new/plan`，复用 PhaseSummary 调整模式布局：
- **学习档案**：表格展示，支持编辑（🖉/✕✓/↩ 三态 + dirty 星号）
- **学习计划**：阶段导航滑轨 + 章节画布 + 详情栏 + 编辑表格（拖拽排序/添加阶段/锁定模式）
- 底部按钮：
  - **调整方案**（→ `/courses/new?adjust=true`）：`isDirty` 时绿色，否则灰色
  - **开始学习**：`!isDirty` 时绿色 → 生成 mock 课程 + 跳转 `/courses`
  - 逻辑互斥：一个绿色时另一个灰色

### mock 课程生成

点击「开始学习」时：
- 从 `profileValues` 提取信息生成 `CourseSessionListItem`
- mock 课程 ID 以 `mock-` 开头，删除走内存移除不调 API
- `CoursesPage` 合并 API 结果 + mock 课程一起显示

**mock 数据存储文件**：`apps/web/src/data/mockCourses.ts`

该文件是**内存级存储**，提供 3 个导出函数：

| 函数 | 作用 |
|------|------|
| `addMockCourse(course)` | 添加一门 mock 课程到数组 |
| `getMockCourses()` | 获取全部 mock 课程 |
| `removeMockCourse(id)` | 按 ID 删除（仅匹配 `mock-` 前缀） |

**关键约束**：
- 数据仅在当前浏览器会话中保留，刷新页面后丢失
- 不接入后端，不写入任何持久化存储
- **未来接入真实 API 后，直接删除整个 `data/mockCourses.ts` 文件，并移除 `CoursesPage` 和 `SetupContinuationPages` 中对它的引用即可**

### 涉及文件

| 文件 | 角色 |
|------|------|
| `App.tsx` | 路由精简 5→2 |
| `AppShell.tsx` | setupSteps 改为 2 步 |
| `SetupPages.tsx` | BuildProfilePage：4 子步骤 + adjust 模式 + 上传弹窗 |
| `SetupContinuationPages.tsx` | ConfirmPlanPage：复用调整模式布局 + mock 课程生成（第 675-700 行附近，`handleStartLearning`） |
| `CoursesPage.tsx` | 合并 API 结果 + `getMockCourses()` 显示；删除 mock 课程时调 `removeMockCourse` |
| `data/mockCourses.ts` | **mock 存储**：3 个导出函数，内存级，刷新丢失，未来接入 API 后整体删除 |
| `styles.css` | 新增 build-profile / material-list-top 等样式 |
