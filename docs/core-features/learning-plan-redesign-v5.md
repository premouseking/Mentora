### 4.2 导航栏设计详解

#### 核心交互：滑轨式滚轮切换

- **无箭头按钮**：不使用 ◀▶ 箭头
- **横向滑轨**：所有阶段排列在一条水平滑轨上
- **滚轮驱动**：鼠标滚轮控制阶段切换
- **最多可见 4 个**：视口内同时最多显示 4 个阶段块

#### 可见窗口与滚动逻辑

设阶段总数为 N（N > 4），可见窗口固定容纳 4 个阶段块。

##### 向右滚动（滚轮向下）

```
初始：[ 1  2  3  4 ] 5  6  ...     选中 = 1（左 1）
滚到2：[ 1  2  3  4 ] 5  6  ...     选中 = 2（左 2）
滚到3：[ 1  2  3  4 ] 5  6  ...     选中 = 3（左 3）
滚到4：[ 2  3  4  5 ] 6  ...        窗口左移 1 位，选中 = 4（左 3）
滚到5：[ 2  3  4  5 ] 6  ...        选中 = 5（左 4）→ 触发位移
滚到6：[ 3  4  5  6 ] ...           窗口左移 1 位，选中 = 6（左 3）
```

**向右滚动规则**：
- 选中在窗口内从最左向右移动，到达窗口内第 3 位（左起第 3 个）时触发窗口左移
- 窗口左移后，选中保持在左起第 3 位

##### 向左滚动（滚轮向上）

```
当前位置：[ 3  4  5  6 ] ...         选中 = 6（左 3）
滚到5：  [ 3  4  5  6 ] ...         选中 = 5（左 2）
滚到4：  [ 3  4  5  6 ] ...         选中 = 4（左 1）
滚到3：  [ 2  3  4  5 ] 6  ...      窗口右移 1 位，选中 = 3（右 3）
滚到2：  [ 2  3  4  5 ] 6  ...      选中 = 2（右 4）→ 触发位移
滚到1：  [ 1  2  3  4 ] 5  ...      窗口右移 1 位，选中 = 1（右 3）
```

**向左滚动规则**：
- 选中在窗口内从右向左移动，到达窗口内右起第 3 位时触发窗口右移
- 窗口右移后，选中保持在右起第 3 位

##### 直观总结

| 滚动方向 | 选中停留位置 | 触发位移条件 |
|----------|-------------|-------------|
| 向右（↓） | 左起第 3 | 选中到达左 4 |
| 向左（↑） | 右起第 3 | 选中到达右 4 |

> 核心体验：滚动的方向不同，选中停留的位置也不同——向右滚最终停在偏右位置，向左滚最终停在偏左位置。

##### 阶段数 ≤ 4 时

窗口无需位移，直接滚动选中即可，无窗口切换逻辑。

#### 阶段项箭头形设计

每个阶段块不采用矩形，而是**斜切平行四边形**——整体向右倾斜，左凹右凸，相邻块在对角线处无缝嵌合：

```
  ╲────────╲╲────────╲╲────────╲╲────────╲
   ╲ 基础   ╲╲  专项   ╲╲  综合   ╲╲  冲刺   ╲
   ╱ 入门   ╱╱  训练   ╱╱  应用   ╱╱         ╱
  ╱────────╱╱────────╱╱────────╱╱────────╱
   ↑        ↑         ↑         ↑
  块1      块2       块3       块4
```

**形状逻辑**：
- 上边和下边分别用 `╲` 和 `╱` 对角线切斜 → 形成向右倾斜的平行四边形
- 左边线由对角线自然形成**凹口**，右边线形成**凸尖**
- 两个块拼接时：`╲╲`（块1右 + 块2左 共用对角线）、`╱╱`（底部同理）
- 共用对角线即是嵌合缝，视觉上无缝连接

**视觉特征**：
- **左侧凹入**：接受前一个块的凸出部分
- **右侧凸出**：嵌入下一个块的凹入部分
- **连续流动**：多个块拼接后形成一条向右倾斜的连续箭头链
- **选中态**：当前选中块高亮（如颜色加深或边框加粗），其余为灰态

## 六、阶段类型池（8 种）与 AI 分阶段 prompt 策略

| 类型 | 适用场景 |
|------|---------|
| 基础入门 | 零基础起步，核心概念和基本方法 |
| 知识梳理 | 系统学习教材，按章节深入 |
| 专项训练 | 突破薄弱点，题型分类训练 |
| 综合应用 | 跨知识点实战，真题/项目 |
| 实战演练 | 限时模拟，成套练习 |
| 查漏补缺 | 错题回看，高频错误强化 |
| 考前冲刺 | 考点回顾，临场策略 |
| 拓展提升 | 课外延伸，竞赛/前沿方向 |

### AI 分阶段策略

Prompt 需引导 AI 根据不同学习档案调整阶段分配：

| 档案特征 | 建议阶段序列 | 权重倾向 |
|----------|-------------|---------|
| 零基础冲刺考试 | 基础入门 → 专项训练 → 考前冲刺 | 60/25/15 |
| 有基础冲刺考试 | 知识梳理 → 专项训练 → 综合应用 → 考前冲刺 | 20/30/30/20 |
| 系统学习无截止 | 基础入门 → 知识梳理 → 综合应用 → 拓展提升 | 20/40/25/15 |
| 快速入门 | 基础入门 → 专项训练 → 实战演练 | 50/30/20 |
| 考前突击 | 查漏补缺 → 考前冲刺 | 40/60 |
| 补弱专项 | 专项训练 → 综合应用 | 70/30 |

**prompt 策略规则**：
1. **先看截止日期**：紧迫 → 压缩基础，加大冲刺/查漏
2. **再看当前基础**：零基础 → 基础入门权重高，不得跳过
3. **再看学习目标**：考试类 → 必含考前冲刺；兴趣类 → 可含拓展提升
4. **最后看时间预算**：时间少 → 减少阶段数（3-4 个）；时间充足 → 5 个
